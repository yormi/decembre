# nutrition/strategy/ — farm-level nutrition strategy

Cross-subproject, farm-level priority framing — above the
per-subproject spec tree. What's *binding*, what we're aiming
for, in what order.

- `problem.md` — the current binding diagnosis (what's capping yield).
- `goal.md` — target-state subgoals, priority order, levers, monitoring.

Lives here (not in a subproject's `doc/`) when it spans multiple
subprojects or sets cross-cutting priority. Single-subproject
analysis stays in that subproject. Decision trails → `learnings/`;
testable rules → `spec.md`.
